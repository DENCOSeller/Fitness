# DENCO Health — Стратегия разработки

Персональный health-трекер. Веб-приложение, один пользователь, браузер на телефоне и десктопе.

---

## Архитектура

```
┌────────────────────────────────────────┐
│  Браузер (телефон / десктоп)           │
│  └── Next.js (React + Server Actions)  │
├────────────────────────────────────────┤
│  Next.js сервер (Beget VPS)            │
│  ├── API Routes / Server Actions       │
│  ├── Prisma ORM → PostgreSQL           │
│  ├── Claude API (AI + Vision)          │
│  └── Загрузка файлов (фото, XML)      │
├────────────────────────────────────────┤
│  PostgreSQL (Beget VPS)                │
└────────────────────────────────────────┘
```

Одно приложение, один деплой. Next.js — и фронтенд, и бэкенд.

### Ключевые решения

| Решение | Причина |
|---------|---------|
| **Next.js (App Router)** | Фронтенд + API в одном проекте. Server Actions упрощают работу с данными. SSR для быстрой загрузки |
| **PostgreSQL + Prisma** | Веб-приложение = серверная БД. Prisma даёт типизацию, миграции, удобные запросы |
| **Claude API (Vision)** | AI-рекомендации + парсинг скриншотов Picooc + анализ фото еды |
| **Без отдельного бэкенда** | Next.js API Routes заменяют Express/Fastify. Один процесс, проще деплой |
| **Tailwind CSS** | Адаптивный дизайн из коробки, быстрая вёрстка, mobile-first |

## Модули

### 1. Ежедневный чек-ин

- Самочувствие, сон, стресс, энергия (1–10, слайдеры)
- Свободная заметка
- Один чек-ин в день, редактируемый до конца дня
- История чек-инов с фильтрацией по датам

### 2. Журнал тренировок

- Тип тренировки (силовая, кардио, растяжка, своё)
- Упражнения: название, подходы × повторения × вес
- Длительность, заметки
- Библиотека упражнений (растёт по мере использования)
- Шаблоны тренировок (сохранить и переиспользовать)
- Таймер отдыха между подходами
- История прогресса по каждому упражнению

### 3. Трекер питания

- Приём пищи (завтрак / обед / ужин / перекус)
- Текстовое описание + фото
- Фото загружается на сервер, хранится на диске VPS
- AI-анализ фото — по кнопке, не автоматически (экономия токенов)

### 4. Apple Health (импорт XML)

**Как работает:**
- В приложении Health на iPhone → профиль → «Экспортировать все данные»
- Скачивается ZIP с файлом `export.xml`
- Загружаешь ZIP в DENCO Health через форму
- Сервер парсит XML, извлекает нужные данные, сохраняет в БД

**Что извлекаем:**
- Шаги (daily total)
- Активные калории
- Пульс покоя
- Данные сна (длительность)
- Тренировки (тип, длительность, калории)

**Важные нюансы:**
- Файл `export.xml` может весить 200–500 МБ при активном использовании Apple Watch
- Парсить DOM-парсером нельзя — не хватит памяти
- Нужен потоковый SAX-парсер (`sax-js` или `fast-xml-parser` в stream mode)
- Парсинг выполняется в фоне, прогресс отображается пользователю
- Повторный импорт — upsert по дате, дубликаты не создаются

### 5. Picooc (скриншот → Claude Vision)

**Как работает:**
- Взвесился → открыл приложение Picooc → сделал скриншот экрана с данными
- Загружаешь скриншот в DENCO Health
- Claude Vision API считывает с экрана: вес, % жира, мышечная масса, вода, BMI
- Данные сохраняются в БД автоматически

**Логика:**
- Промпт для Claude: структурированный, с примером ожидаемого JSON-ответа
- Валидация: вес 30–300 кг, жир 3–60%, мышцы 20–60 кг — отсечь галлюцинации
- Показать распознанные данные для подтверждения перед сохранением
- Стоимость одного Vision-запроса: ~$0.01–0.03

### 6. AI-рекомендации (Claude API)

**Что делает:**
- Анализ трендов (чек-ин данные за 7–30 дней)
- Рекомендации по тренировкам на основе прогресса и самочувствия
- Оценка питания по фото и описанию
- Еженедельный дайджест

**Логика:**
- Текстовые запросы через Claude API (модель: claude-sonnet-4-6 для экономии, claude-opus-4-6 для еженедельных отчётов)
- Контекст формируется на сервере из БД: последние 7–30 дней данных → JSON → промпт
- Кэширование: дневная рекомендация генерируется один раз и сохраняется в БД

**Бюджет:**
- Sonnet (ежедневные советы, парсинг Picooc): ~$5–10/мес
- Opus (еженедельные отчёты): ~$5–10/мес
- Vision (фото еды, ~2 фото/день): ~$10–20/мес
- **Итого: $20–40/мес** при активном использовании

### 7. Графики и прогресс

- Вес / жир / мышцы — линейный график (данные из Picooc)
- Объёмы тренировок — столбчатый по неделям
- Тренды чек-ина — самочувствие / сон / стресс / энергия
- Прогресс по упражнениям — кривая веса
- Шаги / калории / пульс — из Apple Health
- Библиотека: **Recharts** (React-нативная, простая, хорошо работает с SSR)

## Схема базы данных

```sql
-- Чек-ины
check_ins (id, date UNIQUE, wellbeing, sleep, stress, energy, note, created_at, updated_at)

-- Тренировки
exercises (id, name, muscle_group, created_at)
workouts (id, date, type, duration_min, note, created_at)
workout_sets (id, workout_id FK, exercise_id FK, set_order, reps, weight, created_at)

-- Шаблоны
workout_templates (id, name, created_at)
template_exercises (id, template_id FK, exercise_id FK, sets, reps, weight, sort_order)

-- Питание
meals (id, date, meal_type, description, photo_path, ai_analysis, note, created_at)

-- Тело (из Picooc скриншотов)
body_metrics (id, date, weight, body_fat_pct, muscle_mass, bmi, water_pct, screenshot_path, created_at)

-- Apple Health (импорт из XML)
health_daily (id, date UNIQUE, steps, active_calories, resting_hr, sleep_hours, created_at, updated_at)
health_workouts (id, date, type, duration_min, calories, source, created_at)

-- AI
ai_insights (id, date, type, model, prompt_tokens, completion_tokens, content, created_at)
```

## Авторизация

Один пользователь — максимально простая схема:

- Пароль задаётся через переменную окружения `AUTH_PASSWORD`
- При первом входе: форма с паролем → cookie с HTTP-only сессией
- Middleware в Next.js проверяет cookie на каждом запросе
- Без OAuth, без регистрации, без JWT-токенов

## Деплой (Beget VPS)

```
Nginx (443, SSL)
  └── reverse proxy → localhost:3000
        └── Next.js (PM2)
              └── PostgreSQL (localhost:5432)
```

- **Next.js**: `npm run build` → `pm2 start npm --name denco-health -- start`
- **SSL**: Let's Encrypt через certbot
- **Домен**: поддомен типа `health.denco.store` (или любой)
- **Фото**: хранятся в `/root/Fitness/public/uploads/`, раздаются Next.js
- **Бэкап**: `cron` → `pg_dump` ежедневно → хранить последние 30 дней

## Риски

| Риск | Митигация |
|------|-----------|
| Apple Health XML слишком большой (500+ МБ) | Потоковый SAX-парсер, лимит по датам при импорте (например, последние 6 месяцев) |
| Claude неточно читает скриншот Picooc | Валидация диапазонов + показать данные для подтверждения перед сохранением |
| Beget VPS мало RAM для Next.js + PostgreSQL | Next.js standalone build (~100 МБ RAM). Если мало — увеличить тариф |
| Фото еды заполняют диск | Сжимать при загрузке (sharp), ограничить размер до 500 КБ |
