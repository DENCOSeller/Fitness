# DENCO Health — Разбивка на куски

Каждый кусок — независимая единица работы. Порядок: от фундамента к фичам.

---

## Chunk 1: Инициализация проекта

**Что делаем:**
Создаём Next.js проект с базовыми зависимостями. Настраиваем TypeScript, Tailwind, структуру папок. Без функциональности — только каркас, который запускается.

**Файлы:**
- `package.json`
- `next.config.ts`
- `tsconfig.json`
- `tailwind.config.ts`
- `app/layout.tsx` (корневой layout: шрифт, meta, Tailwind)
- `app/globals.css`
- `app/page.tsx` (заглушка)
- `.env.example`
- `.gitignore`

**Критерии готовности:**
- `npm run dev` запускается без ошибок
- Открывается в браузере на localhost:3000
- Tailwind работает (стили применяются)
- TypeScript компилируется без ошибок

**Зависимости:** нет

## Chunk 2: База данных (Prisma + PostgreSQL)

**Что делаем:**
Подключаем Prisma к PostgreSQL на Beget. Создаём полную схему БД (все таблицы). Выполняем первую миграцию. Создаём синглтон Prisma-клиента.

**Файлы:**
- `prisma/schema.prisma` (все модели: check_ins, exercises, workouts, workout_sets, workout_templates, template_exercises, meals, body_metrics, health_daily, health_workouts, ai_insights)
- `lib/db.ts` (Prisma client singleton)
- `.env` (DATABASE_URL)

**Критерии готовности:**
- `npx prisma migrate dev` выполняется успешно
- `npx prisma studio` показывает все таблицы
- Prisma client импортируется и работает из серверного кода

**Зависимости:** Chunk 1

## Chunk 3: Авторизация

**Что делаем:**
Простая авторизация по паролю. Страница логина, middleware для проверки сессии, HTTP-only cookie.

**Файлы:**
- `lib/auth.ts` (проверка пароля, создание/валидация сессии, cookie)
- `app/(auth)/login/page.tsx` (форма логина)
- `app/(auth)/login/actions.ts` (Server Action: проверить пароль, установить cookie)
- `middleware.ts` (проверка cookie, редирект на /login если нет сессии)
- `.env` (AUTH_PASSWORD, SESSION_SECRET)

**Критерии готовности:**
- Без cookie — редирект на /login
- Правильный пароль → cookie → редирект на /
- Неправильный пароль → сообщение об ошибке
- Cookie HTTP-only, Secure, SameSite=Strict

**Зависимости:** Chunk 1

## Chunk 13: Импорт Apple Health XML

**Что делаем:**
Загрузка ZIP-файла экспорта Apple Health. Потоковый парсинг XML. Извлечение шагов, калорий, пульса, сна. Сохранение в БД (upsert по дате).

**Файлы:**
- `app/(dashboard)/health/page.tsx` (форма загрузки + статус импорта + данные)
- `app/api/health-import/route.ts` (API Route: приём ZIP, распаковка, потоковый парсинг)
- `lib/health-import.ts` (SAX-парсер для Apple Health XML, маппинг типов данных)

**Критерии готовности:**
- Загрузка ZIP файла через форму
- Распаковка, нахождение `export.xml`
- Потоковый парсинг (SAX) — не падает на файлах 500+ МБ
- Извлечение: шаги, активные калории, пульс покоя, длительность сна
- Upsert в `health_daily` по дате (повторный импорт не создаёт дубликатов)
- Извлечение тренировок в `health_workouts`
- Прогресс-бар или лог парсинга
- Показ количества импортированных записей

**Зависимости:** Chunk 2, Chunk 4
